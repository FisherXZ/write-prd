# Contributing

## What to contribute

- **Improvements to SKILL.md** — better interview questions, stronger template sections, new principles
- **Example PRDs** — real (anonymized) or synthetic PRDs generated by the skill, added to `examples/`
- **Platform support** — skill files for AI agents not yet listed in the platform table

## How

1. Fork the repo
2. Make changes to `SKILL.md` (the source of truth)
3. Copy updated `SKILL.md` to all platform dirs: `.cursor/`, `.gemini/`, `.codex/`, `.opencode/`, `.kiro/`
4. Open a PR with a brief description of what changed and why

## Guidelines

- The template structure in `SKILL.md` should stay stable — it's what users learn to expect
- New sections or principles should come with a rationale
- Example PRDs should be self-contained markdown files in `examples/`

## Reporting issues

Open a GitHub issue with: what you asked the skill to do, what you got, what you expected.
